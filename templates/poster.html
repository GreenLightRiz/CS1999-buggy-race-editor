Why Us?

This is the newest and most up-and-coming buggy editor on the market right now! There are multiple options with which this stunning website can provide for you:

  • Home - The main menu for this website! Welcome!
  • Create Buggy - Here, you can customise your very own buggy, and it saves itself as a JSON file.
  • Your Buggy Specifications - The choices you made for your buggy are displayed in a table. Feel free to make as many buggies as you wish, and in this page you can edit them as you please! Unfortunately, because the buggies have gained somewhat of a sentience and they refuse to be deleted, they're there forever. Enjoy!
  • Get Buggy JSON - The buggy's information is put into a JSON file, which you are free to download.
  • Information - The information relating to each part of what a buggy does is conveniently located here!
  • Poster - takes you to this page!


    • A task-by-task explanation of my process in making this website

    PHASE 0

    0-GET Get the source code

    I installed VSCode - the IDE used for this module. To get the source code, I logged into the buggy race server, linked my account to my Github account, and forked the template to my repository.

    Then, after this, I downloaded the VSCode workspace configuration file, opened it with VSCode (and installing any missing plugins/packages in the process).

    0-RUN Get app running and view it in a browser

    Flask is the python library used for handling computation in the webserver. It was already installed in the teaching server, and I ran "python3 app.py" in the terminal to deploy the webserver

    0-CHANGE Make a change to a template and see it appear

    I experimented with making changes to template by changing around the order of written text and adding buttons.


    PHASE 1

    1-TEMPLATE Add a new template to the app

    I added the new templates "info" and "poster", which gave information about things to add to each buggy and a page to display my poster respectively

    1-ADD Add more data to the form

    I needed to add more data to the form, so I added in options for the flag's primary colour, the flag's secondary colour, the flag pattern, the type of tyres, the number of tyres, the type of armour, and primary motive power.

    I used a range of ways to give the user an understanding of the options - such as a colour picker and a list of options in some cases.

    1-VALID Add basic data validation

    For the number of wheels and tyres, they could not be odd and they had to be numbers. As such, I utilised "isdigit()" and modulus to ensure that any cases where anything that was not an even number 4 or above would be invalid.

    For the colours of the flags, I used a colour picker so the user could easily pick out the colour of their choice, as validating each typed in input would have been too difficult to validate. The flag colours are stored as hexadecimal codes.

    The flag pattern, types of tyres, types of armour, and primary motive powers use the "select option" function, which has the user choose an option from a list.

    I wanted to include a price limit, but I could not find anywhere on the buggy server websites what the maximum cost could be, so I did not implement this feature.

    1-STYLE Style your editor just how you like it

    I used CSS to change the design of the website to my liking. The buttons have a small animation where they change colour to white when they are hovered on, and the navigation bar options have a line that is created upon hovering on it.

    Some buttons - namely, the "edit a buggy" buttons would not display the hover animation.


    PHASE 2

    2-EDIT Edit the record by loading its current values into the form

    I used an "IF" statement so that if the user chose an option, next time they chose to create a buggy it would default to whatever they last chose.

    However, I find that it has stopped working somewhere along my project. It now constantly defaults to "Please Select", though I feel as though this is better, because it stays consistent in being as user-friendly as possible.

    2-FORM Make the form better

    I gave brief explanations to how creation of buggies worked, and colour-coded if something went wrong or right (if a buggy was created wrong or a piece of data was missing, it would show an error message with a red background colour around the text, and green for if everything was successfully created)

    I attempted to include pictures and videos, though my website did not recognise them.

    2-COST Calculate and save the game cost of the buggy

    I created a variable called "total_cost" and set it to 0 at first, but every time a certain item was selected - for instance, if knobbly tyres were selected) it would add the cost on to it for each choice made and outputted the total in the msg variable.

    2-RULES Add validation according to the game rules

    I made sure that there was a minimum of 4 tyres and wheels, and if a number less than 4 was inputted it would output an error. As well as this, anything that wasn't an even number would also output an error.

    I also got rid of any need for validation with the flag colours, as the input text boxes were replaced with colour pickers. As well as this, the "select" options would output an error if any of them were left on the "please select" option.


    PHASE 3

    3-ENV Switch between dev and production environments

    I switched between dev and production mode by doing FLASK_ENV=development and FLASK_ENV=production respectively.

    3-AUTOFILL Add auto-fill to the edit forms

    I attempted to make a button that would automatically fill in the entries with default values, though it kept making an error where it generated empty tables no matter what I tried. Because of time constraints, I had to scrap the "default options" button.

    3-MULTI Allow different buggies to be created

    I slightly changed the SQL so that it would create a new table for new buggies, rather than updating the same table. Every time a buggy is created, it adds another table of buggies, each with their own ID.

    I also implemented an edit button, so that an end user could change around their buggies as they please.

    3-DEL Allow buggies to be deleted

    I attempted to create a button that would allow buggies to be deleted, though due to time constraints I could not complete this feature and thus it was removed.
